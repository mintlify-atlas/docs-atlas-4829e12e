You can add projects to your portfolio by editing the `projects` array in `src/data/projects.ts`. Each project is type-safe and follows a consistent structure.

## Project data structure

The `ProjectItem` interface is defined in `src/types/index.ts:17-24`:

```typescript index.ts
export interface ProjectItem {
  title: string;        // Project name
  description: string;  // Brief project description
  techStack: string[];  // Array of technologies used
  link?: string;        // Optional live demo URL
  github?: string;      // Optional GitHub repository URL
  features?: string[];  // Optional list of key features
}
```

<Note>
Fields marked with `?` are optional. At minimum, you need `title`, `description`, and `techStack` to create a valid project entry.
</Note>

## Adding a new project

Follow these steps to add a project to your showcase.

<Steps>
  <Step title="Open the projects file">
    Navigate to `src/data/projects.ts` in your code editor.
  </Step>
  
  <Step title="Add links to constants (if needed)">
    If your project has a live demo or GitHub repository, first add the URLs to `src/lib/constants.ts` in the `ExternalLinks` object.
  </Step>
  
  <Step title="Create the project object">
    Add a new object to the `projects` array following the structure shown below.
  </Step>
  
  <Step title="Save and preview">
    Save the file and check your browser. The new project should appear automatically with hot reloading.
  </Step>
</Steps>

### Basic example

Here's a minimal project with required fields only:

```typescript projects.ts
{
  title: "Task Manager Pro",
  description: "A collaborative task management application with real-time updates, drag-and-drop functionality, and team workspaces.",
  techStack: ["Next.js", "TypeScript", "Prisma", "PostgreSQL", "Tailwind CSS"],

```

### Complete example with all fields

```typescript projects.ts
{
  title: "E-Commerce Platform",
  description: "Full-stack e-commerce solution with payment processing, inventory management, and admin dashboard. Supports multiple payment gateways and real-time order tracking.",
  techStack: ["Next.js 15", "TypeScript", "Stripe", "Prisma", "PostgreSQL", "Redis"],
```

<Tip>
When adding links, use the `ExternalLinks` object from `src/lib/constants.ts` rather than hardcoding URLs. This keeps all links centralized and type-safe.
</Tip>

## Writing effective descriptions

Your project descriptions should be clear, concise, and compelling.

### Description best practices

<CardGroup cols={2}>
  <Card title="Lead with impact">
    Start with the most impressive metric or unique value proposition. "100,000+ active users" or "30+ partner psychologists" immediately establishes scale.
  </Card>
  
  <Card title="Be specific about features">
    Instead of "user authentication," say "JWT token rotation with automatic refresh" to showcase deeper technical knowledge.
  </Card>
  
  <Card title="Include technical details">
    Mention specific libraries, algorithms, or patterns you implemented (e.g., "X.509 certificate parsing," "Server Actions as RPC").
  </Card>
  
  <Card title="Keep it scannable">
    Aim for 2-4 sentences. Readers should grasp the project's scope within 5-10 seconds of reading.
  </Card>
</CardGroup>

### Description length guidelines

<CodeGroup>
```text Too brief
"A CRM application with authentication."
// Missing: tech details, scale, unique features
```

```text Well-balanced
"Full-stack CRM with FastAPI backend and Next.js 15 frontend. Features citizen data integration (ASAN), Jitsi video meetings, and JWT auth."
// Good: specific technologies, key features, architecture
```

```text Too verbose
"This is a comprehensive customer relationship management system that I built from scratch using modern technologies. It has a backend written in FastAPI which handles all the API requests, and a frontend built with Next.js 15 that provides a great user experience. The system integrates with ASAN for citizen data, includes video calling functionality powered by Jitsi, and implements secure authentication using JSON Web Tokens."
// Too long: excessive detail, repetitive, hard to scan
```
</CodeGroup>

## Selecting your tech stack

List technologies strategically to highlight your skills.

### Tech stack ordering

Order technologies by importance and relevance:

<Steps>
  <Step title="Primary framework first">
    Start with your main framework or language (e.g., "Next.js", "React Native", "FastAPI").
  </Step>
  
  <Step title="Core technologies next">
    Include essential tools like TypeScript, databases, or state management libraries.
  </Step>
  
  <Step title="Notable integrations last">
    End with interesting third-party services or specialized libraries that demonstrate breadth.
  </Step>
</Steps>

### Example tech stack

```typescript projects.ts
techStack: [
  "Next.js 15",           // Primary framework
  "TypeScript",           // Language
  "Server Actions",       // Architecture pattern
  "Zod",                  // Validation library
  "shadcn/ui",           // UI component library
  "TailwindCSS",         // Styling
  "Docker",              // DevOps
]
```

<Note>
Limit your tech stack to 4-8 items. Too few suggests a simple project, too many becomes overwhelming and dilutes your message.
</Note>

## Adding project links

Links to live demos and GitHub repositories enhance credibility.

### Adding links to constants

Before referencing links in your project, add them to `src/lib/constants.ts:1-17`:

```typescript constants.ts
export const ExternalLinks = {
  // ... existing links
  
  // Add your project links
  MyProjectDemo: "https://myproject.vercel.app",
  MyProjectGithub: "https://github.com/username/my-project",
} as const;
```

### Using links in projects

Reference the links in your project object:

```typescript projects.ts
import { ExternalLinks } from "@/lib/constants";

{
  title: "My Awesome Project",
  description: "...",
  techStack: ["Next.js", "TypeScript"],
  link: ExternalLinks.MyProjectDemo,
  github: ExternalLinks.MyProjectGithub,
}
```

<Tip>
If you only have a GitHub repository without a live demo, you can omit the `link` field. Similarly, omit `github` for proprietary projects.
</Tip>

## Project ordering

Projects are displayed in the order they appear in the array.

### Recommended ordering strategy

<Steps>
  <Step title="Most impressive first">
    Place your best or most recent projects at the top. These get the most visibility.
  </Step>
  
  <Step title="Group by type (optional)">
    You can organize by category: commercial projects first, then personal projects, then demos.
  </Step>
  
  <Step title="Balance variety">
    Alternate between different tech stacks to showcase breadth (e.g., don't list all React projects together).
  </Step>
</Steps>

### Example ordering

## Using the features field

The optional `features` array lets you highlight key capabilities.

### When to use features

Use the `features` field when:

- Your project has 4+ distinct capabilities worth highlighting
- The description would become too long if you listed all features inline
- You want to make specific technical achievements scannable

## Type safety validation

TypeScript ensures your project data is valid.

### Common type errors

<AccordionGroup>
  <Accordion title="Missing required fields">
    ```typescript
    // ❌ Error: Property 'description' is missing
    {
      title: "My Project",
      techStack: ["React"],
    }
    
    // ✅ Correct: All required fields present
    {
      title: "My Project",
      description: "A React application for...",
      techStack: ["React"],
    }
    ```
  </Accordion>
  
  <Accordion title="Wrong field types">
    ```typescript
    // ❌ Error: techStack must be an array
    {
      title: "My Project",
      description: "...",
      techStack: "React, TypeScript",
    }
    
    // ✅ Correct: techStack is an array of strings
    {
      title: "My Project",
      description: "...",
      techStack: ["React", "TypeScript"],
    }
    ```
  </Accordion>
  
  <Accordion title="Invalid link references">
    ```typescript
    // ❌ Error: Property 'NonExistentLink' does not exist
    {
      title: "My Project",
      description: "...",
      techStack: ["React"],
      link: ExternalLinks.NonExistentLink,
    }
    
    // ✅ Correct: Link exists in ExternalLinks
    {
      title: "My Project",
      description: "...",
      techStack: ["React"],
      link: ExternalLinks.MyProjectDemo,
    }
    ```
  </Accordion>
</AccordionGroup>

<Tip>
Run `npm run type-check` or `npm run build` to validate all projects before deploying. TypeScript will catch any errors at compile time.
</Tip>
