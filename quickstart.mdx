## Prerequisites

Before you begin, make sure you have the following installed on your machine:

- **Node.js** 20.x or higher
- A package manager: **pnpm** (recommended), npm, yarn, or bun
- **Git** for version control

<Note>
  This project was built with pnpm, but all package managers are supported. The examples below show multiple options.
</Note>

## Get the code

<Steps>
  <Step title="Clone the repository">
    First, clone the portfolio repository to your local machine:
    
    <CodeGroup>
    ```bash SSH
    git clone git@github.com:ahm3tj4f/portfolio.git
    cd portfolio
    ```
    
    ```bash HTTPS
    git clone https://github.com/ahm3tj4f/portfolio.git
    cd portfolio
    ```
    </CodeGroup>
    
    <Info>
      Replace the repository URL with your own fork if you've already forked the project.
    </Info>
  </Step>
  
  <Step title="Install dependencies">
    Install all required packages using your preferred package manager:
    
    <CodeGroup>
    ```bash pnpm
    pnpm install
    ```
    
    ```bash npm
    npm install
    ```
    
    ```bash yarn
    yarn install
    ```
    
    ```bash bun
    bun install
    ```
    </CodeGroup>
    
    This will install all dependencies listed in `package.json`, including:
    - Next.js framework and React
    - UI libraries (Radix UI, Lucide icons)
    - 3D graphics tools (Three.js, React Three Fiber)
    - MDX processing for blog posts
    - Development tools (TypeScript, ESLint, Husky)
  </Step>
  
  <Step title="Start the development server">
    Run the local development server:
    
    <CodeGroup>
    ```bash pnpm
    pnpm dev
    ```
    
    ```bash npm
    npm run dev
    ```
    
    ```bash yarn
    yarn dev
    ```
    
    ```bash bun
    bun dev
    ```
    </CodeGroup>
    
    The development server will start on `http://localhost:3000`.
  </Step>
  
  <Step title="Open in your browser">
    Navigate to [http://localhost:3000](http://localhost:3000) to see your portfolio website.
    
    You should see the homepage with:
    - A hero section with the developer's name
    - An about section describing their experience
    - An animated ASCII wave component
    - An approach section detailing their philosophy
    
    <Info>
      The page automatically refreshes when you make changes to the code. Try editing `src/app/page.tsx` to see live updates.
    </Info>
  </Step>
</Steps>

## Available scripts

The portfolio comes with several npm scripts defined in `package.json`:

| Script | Command | Description |
|--------|---------|-------------|
| `dev` | `next dev` | Starts the development server with hot reload |
| `build` | `next build` | Creates an optimized production build |
| `start` | `next start` | Runs the production server (requires build first) |
| `lint` | `eslint` | Runs ESLint to check code quality |

<Note>
  The project includes Husky for Git hooks. A `precommit` hook automatically generates a PDF version of the resume before each commit.
</Note>

## Verify your setup

To ensure everything is working correctly:

<Steps>
  <Step title="Check the homepage">
    Verify that `http://localhost:3000` displays the portfolio homepage with all sections rendering properly.
  </Step>
  
  <Step title="Test hot reload">
    Open `src/app/page.tsx` and modify some text. The browser should automatically refresh with your changes.
  </Step>
  
  <Step title="Run the linter">
    Execute the lint command to verify code quality:
    
    <CodeGroup>
    ```bash pnpm
    pnpm lint
    ```
    
    ```bash npm
    npm run lint
    ```
    </CodeGroup>
  </Step>
  
  <Step title="Build for production">
    Test that the production build works:
    
    <CodeGroup>
    ```bash pnpm
    pnpm build
    pnpm start
    ```
    
    ```bash npm
    npm run build
    npm start
    ```
    </CodeGroup>
    
    The optimized site will be available at `http://localhost:3000`.
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Port 3000 is already in use">
    If you see an error that port 3000 is already in use, you can:
    
    - Stop the other process using port 3000
    - Or run Next.js on a different port:
    
    ```bash
    pnpm dev -p 3001
    ```
  </Accordion>
  
  <Accordion title="Module not found errors">
    If you see module not found errors:
    
    1. Delete `node_modules` and package lock files
    2. Reinstall dependencies: `pnpm install`
    3. Clear Next.js cache: `rm -rf .next`
    4. Restart the dev server
  </Accordion>
  
  <Accordion title="TypeScript errors in editor">
    If your editor shows TypeScript errors:
    
    1. Ensure you have TypeScript installed globally or in the project
    2. Restart your TypeScript server in VS Code: Cmd/Ctrl + Shift + P → "Restart TypeScript Server"
    3. Check that your editor is using the workspace TypeScript version
  </Accordion>
</AccordionGroup>
